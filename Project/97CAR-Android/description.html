Description du projet



Le diagramme de cas d’utilisation ci-dessous illustre les principales fonctionnalités de l’application CarSlim dans le cadre du projet. Il met en évidence à la fois les fonctionnalités initialement disponibles, telles que la gestion des clients et des factures, et celles que j’ai ajoutées ou améliorées. Cela comprend l’intégration complète de la gestion des véhicules (ajout, consultation, modification, suppression), l’enrichissement des écrans de consultation des clients et des factures, ainsi que l’implémentation des opérations d’ajout, de modification et de suppression pour les clients, en réponse aux besoins exprimés dans le cahier des charges.



Dans le cadre de mon projet, cette section présente l’interface de l’application CarSlim à travers une série de captures d’écran, illustrant les fonctionnalités initialement disponibles, ainsi que celles que j’ai ajoutées ou modifiées pour répondre aux objectifs du projet.

1. Clients


Liste des clients enregistrés

Affiche la liste complète des clients enregistrés dans le système. ​

Routes associées : GET /clients





Ajout d’un nouveau client

Permet de créer un nouveau client via un formulaire. ​

Routes associées : POST /clients







Détail d’un client

Affiche les informations détaillées d’un client spécifique. ​

Routes associées : GET /clients/{id}





Modification d’un client

Permet de modifier les informations d’un client existant. ​

Routes associées : PUT /putclients/{id}







Suppression d’un client

Permet de supprimer un client du système. ​

Routes associées : DELETE /dropclients/{id}





2. Voitures



Ajout d’une nouvelle voiture

Permet d’enregistrer une nouvelle voiture associée à un client. ​

Routes associées : POST /voitures





Détail d’une voiture

Affiche les détails d’une voiture spécifique. ​

Routes associées : GET /voitures/{id}





Modification d’une voiture

Permet de modifier les informations d’une voiture existante. ​

Routes associées : PUT /voitures/{id}







Suppression d’une voiture

Permet de supprimer une voiture du système. ​

Routes associées : DELETE /voitures/{id}





3. Factures


Détail d’une facture

Affiche les informations détaillées d’une facture spécifique. ​

Routes associées : DGET /factures/{id}
