Contexte du projet



Dans le cadre de mon stage en tant que développeur informatique, ma mission consiste à développer un système de gestion simple et performant afin de faciliter l’organisation des opérations quotidiennes du garage. L’objectif est de poursuivre le développement de l’application CarSlim (application Android) destinée aux commerciaux, en y ajoutant de nouvelles fonctionnalités et en assurant une communication fluide avec l’API du backend.


Connexion aux données :

  • Mise en place d’une communication entre l’application Android et la base de données via une API REST.
  • L’API est développée en PHP avec le micro-framework Slim.
  • La communication entre l’application Android et l’API se fait à l’aide de Retrofit (bibliothèque HTTP pour Kotlin).

Fonctionnalités de l’application Android :


Une partie des fonctionnalités a déjà été mise en place.

La mission consiste à :
  • Tester celles déjà développées.
  • Développer les fonctionnalités manquantes, notamment :
Intégration des routes API REST :
    • GET /voitures Récupérer la liste des voitures.

    • GET /voitures/{id} Obtenir les détails d’une voiture via son identifiant.

    • POST /voitures Ajouter une nouvelle voiture à partir des données JSON envoyées par l'application Android.

    Fonctions à ajouter dans l’interface utilisateur :
  • Modification et suppression des clients.
  • Ajout, modification et suppression des voitures.

Environnement technique :

  • Langages et Frameworks : Android : Kotlin + Jetpack Compose API : PHP (Slim Framework)
  • Base de données : PostgreSQL
  • Outils de virtualisation : Docker et Hyper-V
  • Contrôle de version : Git et Github
  • Architecture logicielle : MVVM et REST
  • Méthodologie : Approche agile pour les évolutions fonctionnelles

Documentation :

  • Rédaction de guides techniques pour faciliter la prise en main, la maintenance et l’évolution du projet.
